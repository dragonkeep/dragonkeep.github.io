### 请求生命周期
该系统中几乎每个PHP文件都引用了`includes/common.php`依赖文件
```
Request
  │
  ▼
includes/common.php 加载
  │
  ├─[1]  include base.php              ← CC防御配置 (define CC_Defender)
  ├─[2]  session_start()               ← 启动 PHP Session
  ├─[3]  autoloader + register         ← 类自动加载
  ├─[4]  CC_Defender 检查              ← CC攻击防御（cookie校验）
  ├─[5]  加载 360safe WAF              ← 请求参数过滤
  ├─[6]  require config.php            ← 加载数据库配置
  ├─[7]  new PdoHelper($dbconfig)      ← 初始化 $DB
  ├─[8]  $CACHE->pre_fetch()           ← 从 pre_cache 加载系统配置
  ├─[9]  define('SYS_KEY', ...)        ← 定义系统密钥
  ├─[10] $password_hash = $conf['syskey']  ← 密码哈希=系统密钥
  ├─[11] include function.php          ← daddslashes(), authcode() 等
  ├─[12] include member.php            ← 【鉴权核心】解析 Cookie Token
  ├─[13] 分站域名匹配                   ← 多租户域名绑定
  ├─[14] cookiesid 初始化              ← 访客标识
  └─[15] 业务文件继续执行
```


### 白盒审计
**未授权使用AI服务**
在`admin/ai_api.php` 中使用变量`$islogin`进行鉴权用户是否登录，默认硬编码为1。正常 admin 页面会重定向到 `login.php`，但此端点直接被访问，若配置了 DeepSeek API Key，则可未授权调用 AI 服务。
![](assets/小杰云商城系统/Pasted_image_20260612164650.png)

**明文存储系统密码**
在`install/index.php`下，系统默认安装将明文密码存储到数据库当中。
![](assets/小杰云商城系统/Pasted_image_20260612165153.png)
在`admin/login.php`中，我们也可以看到系统直接使用数据库中明文密码与用户提交的密码进行对比判断，这为后续的攻击链埋下伏笔。
![](assets/小杰云商城系统/Pasted_image_20260612165329.png)

**未授权LIKE注入**
在`newApi.php`当中，使用了`if (isset($_POST['user']) && isset($_POST['pass']))`的IF语句进行判断，实际攻击者可以直接两个参数都不传递就绕过认证。
![](assets/小杰云商城系统/Pasted_image_20260612165958.png)
kw参数存在LIKE注入，使用`%`即可查看系统所有商品信息，使用了`daddslashes`函数转义了单引号，且系统默认连接数据库编码为`utf8mb4`，也就杜绝了宽字节注入的可能。
![](assets/小杰云商城系统/Pasted_image_20260612170127.png)
![](assets/小杰云商城系统/Pasted_image_20260612170650.png)
**任意配置覆写**
在`admin/ajax_optimize.php`中，同样存在使用`$islogin`参数认证登录，以及硬编码问题。
![](assets/小杰云商城系统/Pasted_image_20260612170931.png)
在设置do参数为`apply`后，可直接更新数据库中表`pre_config`中的配置项，包括key和value值。
![](assets/小杰云商城系统/Pasted_image_20260612170956.png)
在实际部署数据库后发现并不存在表`pre_config`

![](assets/小杰云商城系统/Pasted_image_20260612171451.png)
这是因为在`includes/lib/PdoHelper.php`中对代码中的表名和实际数据库名进行了前缀替换。
默认将`pre_`前缀替换为`$dbconfig['dbqz']`值，默认在config.php对应是`shua`字符串。
![](assets/小杰云商城系统/Pasted_image_20260612171719.png)
![](assets/小杰云商城系统/Pasted_image_20260612171734.png)
![](assets/小杰云商城系统/Pasted_image_20260612171833.png)
在`shua_config`表中存储着管理员用户名密码，这样我们就可以通过未授权方式将管理员用户名和密码进行覆盖。
![](assets/小杰云商城系统/Pasted_image_20260612172028.png)

**后台命令执行**
管理员后台存在配置CC防护功能，使用PHP代码拼接的方式写入到文件base.php。
![](assets/小杰云商城系统/Pasted_image_20260612172323.png)

base.php在`lib/common.php`中被引用，也就是说，每个PHP文件在加载都需要引用base.php，把恶意代码写入到base.php当中，就能对整个系统进行命令执行。
